Overview

Zveltio uses Better-Auth for complete authentication functionality. Better-Auth is a modern authentication library that provides secure session management, multiple authentication methods, and easy integration with Zveltio.

💡 Key Features
  • Session-based authentication
  • Multiple auth methods (email, OAuth, magic links)
  • Two-factor authentication (2FA)
  • Email verification
  • Password hashing with bcrypt

Authentication Methods

Email/Password

Traditional login with secure password hashing:

# Sign up
POST /api/auth/sign-up/email
{
  "email": "user@example.com",
  "password": "securePassword123",
  "name": "John Doe"
}

# Sign in
POST /api/auth/sign-in/email
{
  "email": "user@example.com",
  "password": "securePassword123"
}

# Sign out
POST /api/auth/sign-out

Magic Links

Passwordless authentication via email:

# Request magic link
POST /api/auth/magic-link/sign-in
{
  "email": "user@example.com"
}

# User receives email with link
# Clicking link automatically signs them in

Setup & Configuration

Environment Variables

# Required - Generate a secure random string
BETTER_AUTH_SECRET=your-64-character-secret-key

# URL of your deployment (no trailing slash)
BETTER_AUTH_URL=https://api.yourdomain.com

# Optional: Trusted origins (comma-separated)
BETTER_AUTH_TRUSTED_ORIGINS=https://studio.yourdomain.com,https://app.yourdomain.com

Password Requirements

Zveltio enforces the following password requirements:

  • Minimum 8 characters
  • At least 1 uppercase letter
  • At least 1 lowercase letter
  • At least 1 number

Session Management

Session Configuration

// Default session settings
{
  sessionMaxAge: 7 * 24 * 60 * 60,  // 7 days in seconds
  sessionUpdateAge: 24 * 60 * 60,     // 24 hours
  cookieCache: 5 * 60            // 5 minutes
}

Session Cookie Settings

Setting Value Purpose
httpOnly true Prevents JavaScript access (XSS protection)
secure true HTTPS only in production
sameSite strict CSRF protection
path / Available on all paths

OAuth Providers

Zveltio supports multiple OAuth providers for social login:

Google

# .env
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

GitHub

# .env
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret

Using OAuth

# Sign in with OAuth
GET /api/auth/sign-in/oauth/google

# Redirects to Google, then back to your app

Two-Factor Authentication

Zveltio supports TOTP-based two-factor authentication:

Enable 2FA

# User enables 2FA in their account settings
# They'll be prompted to scan a QR code with their authenticator app

# Verification
POST /api/auth/two-factor/verify
{
  "code": "123456"
}
⚠️ Important
  • 2FA is required for all admin users
  • Store backup codes in a secure location
  • Use authenticator apps like Google Authenticator or Authy

Security Best Practices

  1. Use strong secrets - Generate a 64-character random string for BETTER_AUTH_SECRET
  2. Enable HTTPS - Always use SSL/TLS in production
  3. Configure CORS - Only allow trusted domains
  4. Enable 2FA - Require for admin accounts
  5. Rotate secrets - Change BETTER_AUTH_SECRET periodically
  6. Monitor sessions - Review active sessions regularly
