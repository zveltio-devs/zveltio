Overview

Webhooks allow your application to receive real-time notifications when events occur in Zveltio. Instead of polling for changes, you can subscribe to events and receive HTTP callbacks.

Events

Event Description
data.create When a new record is created
data.update When a record is updated
data.delete When a record is deleted
auth.user.created When a new user registers
auth.user.updated When user profile is updated
collection.created When a collection is created

Creating Webhooks

POST /api/webhooks
{
  "url": "https://your-server.com/webhook",
  "events": ["data.create", "data.update"],
  "secret": "your-webhook-secret"
}

Security

Webhooks are secured using a shared secret. The secret is included in the X-Webhook-Signature header: 

// Verify webhook signature
const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
  
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}

API Endpoints

Endpoint Method Description
/api/webhooks GET List all webhooks
/api/webhooks POST Create a webhook
/api/webhooks/:id GET Get webhook details
/api/webhooks/:id DELETE Delete a webhook
/api/webhooks/:id/test POST Test a webhook
