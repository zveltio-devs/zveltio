Overview

Dynamic Collections allow you to create database tables through the API without writing SQL. Each collection represents a database table, and fields define the columns.

💡 Zero DDL Downtime
Zveltio creates tables and columns without locking the database, ensuring your application stays available during schema changes.

Creating a Collection

POST /api/collections
{
  "name": "products",
  "fields": [
    { "name": "title", "type": "text", "required": true },
    { "name": "price", "type": "number", "required": true },
    { "name": "description", "type": "richtext" },
    { "name": "in_stock", "type": "boolean", "default": true }
  ]
}

Field Types

Type Description Options
text Short text ( VARCHAR ) maxLength, required
richtext Long text ( TEXT ) required
number Integer or decimal precision, scale, required
boolean true/false default
date Date without time required
datetime Date with time required
email Email address required
url URL required
json JSON object required
file File reference required
reference Relation to another collection collection, required

API Endpoints

Create Collection

POST /api/collections

List Collections

GET /api/collections

Get Collection

GET /api/collections/:name

Update Collection

PATCH /api/collections/:name

Delete Collection

DELETE /api/collections/:name

CRUD Operations

# Create record
POST /api/data/:collection

# List records
GET /api/data/:collection

# Get record
GET /api/data/:collection/:id

# Update record
PATCH /api/data/:collection/:id

# Delete record
DELETE /api/data/:collection/:id

Relationships

Create relationships between collections using reference fields:

{
  "name": "products",
  "fields": [
    { "name": "category", "type": "reference", "reference": "categories" }
  ]
}

Validation

Add validation rules to fields:

{
  "name": "email",
  "type": "email",
  "required": true,
  "unique": true
}
